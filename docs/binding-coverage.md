# Binding Coverage

## Overview

Coverage is tracked against the **complete miniaudio public API** as the denominator.

- **Denominator**: 1,027 core `MA_API` functions (source: `vendor/miniaudio/miniaudio.h`, excluding
  bundled third-party codecs: `ma_dr_*` / `ma_stbvorbis_*`)
- **Inventory source of truth**: `docs/api-inventory.json` (regenerated by
  `pixi run gen-api-inventory`)
- **Binding annotation**: `/* @binds ma_foo, ma_bar */` comment above each shim function in
  `src/native/ma_shim.c` declares which `MA_API` functions that shim wraps
- **Gates**: `pixi run coverage-binding` (structural checker) + `pixi run coverage-shim` (gcov
  line-coverage ≥ 95% on `ma_shim.c`)

## Coverage Levels (DoD scale)

| Level | Meaning |
|-------|---------|
| L0 | Shim function exists in `ma_shim.{h,c}` with `@binds` annotation |
| L1 | L0 + raw free function in `src/miniaudio/_ffi/*_raw.mojo` (Layer 2) |
| L2 | L1 + positive-path test + negative-path test (contract tests in `tests/test_*_binding.mojo`) |
| L3 | L2 + behavioral/stateful tests in `tests/test_*_api.mojo` (RAII API layer) |
| L4 | L3 + integration / multi-component tests |

## Migration Roadmap

Planned migration order (each family follows the three-layer + TDD + gate template):

1. **decoder** ← current (backfilling L3 behavioral tests)
2. **encoder** ← next: first fully gated family migration
3. device / playback
4. engine / sound
5. data_source / ring_buffer
6. converters (resampler / channel / data)
7. nodes / effects / eq
8. resource_manager
9. sync / async / job_queue / log
10. context / vfs

On completion of each family:
- Add shim functions with `@binds` to `src/native/ma_shim.{h,c}`
- Add `_ffi/*_raw.mojo` (Layer 2 free functions)
- Add RAII type under `src/miniaudio/` (Layer 3)
- Add `tests/test_*_binding.mojo` + `tests/test_*_api.mojo`
- Update `docs/coverage-targets.json`: `done=true, dod_level=L2|L3`
- Run `pixi run gate` to green
- Retire matching legacy `src/api/` module and fat-shim section

## Gate Commands

```bash
pixi run gen-api-inventory   # regenerate docs/api-inventory.json (after updating miniaudio vendor)
pixi run coverage-binding    # structural checker (exit 0 = pass)
pixi run coverage-shim       # gcov line-coverage gate on ma_shim.c (exit 0 = ≥95%)
pixi run coverage            # both gates
pixi run gate                # full suite: test + coverage
```

## Coverage Assurance Design

See `docs/binding-architecture.md` for the layer contracts and `docs/binding-test-plan.md`
for the L0–L4 scale definition.

Key invariants enforced by the checker (`tools/check_binding_coverage.py`):
1. Every `ma_shim_*` export has an L2 raw binding AND ≥1 test reference.
2. Every `@binds ma_foo` names a function that actually exists in the 1,027-function
   denominator (typo guard).
3. Families marked `done=true` in `coverage-targets.json` meet their stated DoD level.
